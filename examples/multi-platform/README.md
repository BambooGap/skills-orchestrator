# 多平台同步示例

这个示例展示如何"一次编写，到处运行"——将 Skills 同步到 5 个不同的平台。

## 支持的平台

| 平台 | Target | 输出格式 | 说明 |
|------|--------|---------|------|
| Claude Desktop | agents-md | AGENTS.md | Claude 自动读取 |
| Cursor | cursor | .cursor/rules/*.mdc | Cursor 自动加载 |
| OpenClaw | openclaw | ~/.openclaw/workspace/skills/ | OpenClaw 自动发现 |
| Hermes | hermes | ~/.hermes/skills/ | Hermes Agent 使用 |
| VSCode Copilot | copilot | .github/copilot-instructions.md | Copilot 自动读取 |

## 使用场景

### 场景 1：个人开发者，使用多个 IDE

```bash
# 早上在 Cursor 开发
skills-orchestrator sync cursor

# 下午切换到 VSCode + Copilot
skills-orchestrator sync copilot

# 晚上用 Claude Desktop 代码审查
skills-orchestrator sync agents-md

# Skills 内容完全一致！
```

### 场景 2：团队成员使用不同工具

```bash
# 成员 A: Cursor 用户
skills-orchestrator sync cursor

# 成员 B: VSCode 用户
skills-orchestrator sync copilot

# 成员 C: Claude Desktop 用户
skills-orchestrator sync agents-md

# 成员 D: OpenClaw 用户
skills-orchestrator sync openclaw

# 所有人的 Skills 版本完全一致！
```

## 完整工作流

### 1. 编写 Skills

```bash
skills/
├── debugging.md
├── tdd.md
└── review.md
```

### 2. 生成配置

```bash
skills-orchestrator init --non-interactive
```

### 3. 同步到所有平台

```bash
# 一键同步到所有平台
skills-orchestrator sync agents-md
skills-orchestrator sync cursor
skills-orchestrator sync openclaw
skills-orchestrator sync hermes
skills-orchestrator sync copilot
```

### 4. 验证

```bash
# 检查 Claude Desktop
cat AGENTS.md

# 检查 Cursor
ls .cursor/rules/

# 检查 OpenClaw
ls ~/.openclaw/workspace/skills/

# 检查 Hermes
ls ~/.hermes/skills/

# 检查 Copilot
cat .github/copilot-instructions.md
```

## 同步模式

### 摘要模式（默认）

```bash
skills-orchestrator sync cursor
```

- Required Skills：完整内容
- Passive Skills：仅摘要（表格形式）

**优点**：节省上下文空间  
**适用**：Skills 数量多，大部分不常用

### 全量模式

```bash
skills-orchestrator sync cursor --full
```

- 所有 Skills：完整内容

**优点**：所有 Skills 都可用  
**适用**：Skills 数量少，或需要离线使用

## 最佳实践

### 1. 根据使用频率选择模式

```bash
# 日常开发：摘要模式
skills-orchestrator sync cursor

# 离线环境：全量模式
skills-orchestrator sync cursor --full
```

### 2. 定期同步

```bash
# 每次更新 Skills 后同步
skills-orchestrator build
skills-orchestrator sync cursor
skills-orchestrator sync agents-md
```

### 3. 团队统一版本

```bash
# 提交 skills.lock
git add skills.lock
git commit -m "chore: 更新 skills 版本"
git push

# 团队成员同步
git pull
skills-orchestrator sync cursor
```

## 故障排查

### 问题：Cursor 没有加载 Skills

**解决方案**：
1. 检查 `.cursor/rules/` 目录是否存在
2. 检查文件格式是否为 `.mdc`
3. 重启 Cursor

### 问题：Claude Desktop 没有读取 AGENTS.md

**解决方案**：
1. 确保 `AGENTS.md` 在项目根目录
2. 重启 Claude Desktop
3. 检查文件编码是否为 UTF-8

### 问题：Skills 版本不一致

**解决方案**：
```bash
# 重新同步所有平台
skills-orchestrator sync agents-md --full
skills-orchestrator sync cursor --full
skills-orchestrator sync openclaw --full
```
