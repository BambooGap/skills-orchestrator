# Release Trust

Use this skill when reviewing package, container, or adapter evidence for a SkillOps release.

## Requirements

- Confirm the package version, GitHub Release, and container tag point to the same release line.
- Confirm SBOM and provenance files are generated from the current artifact set.
- Confirm adapter evidence is generated by commands, not manually edited JSON.

## Output

Write a short release-trust note with the artifact paths reviewed.
