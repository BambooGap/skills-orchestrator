# Open-core Boundary

Skills Orchestrator is an open-source SkillOps control plane. The OSS core should remain useful
without a hosted service and should keep producing portable files that other systems can consume.

## OSS Core

The open-source package owns:

- CLI commands: `check`, `schema`, `manifest`, `policy export`, `registry`, `evidence`, `doctor`,
  `adapters`, and `supply-chain`.
- File contracts: JSON Schema, SARIF, CycloneDX, registry JSON, registry diff Markdown, evidence
  manifests, adapter inspection, and package SBOM.
- GitHub Action automation for local CI, SARIF upload, registry diff artifacts, and optional PR
  comments.
- MCP runtime serving for task-scoped skill loading.

## Hosted Or Enterprise Layer

A hosted product may own:

- GitHub App installation management.
- Organization-level artifact ingestion and retention.
- Enterprise dashboard rendering from evidence artifacts.
- Private policy packs, support, rollout playbooks, and customer integrations.

Hosted surfaces should ingest files generated by the OSS CLI. They should not fork resolver,
registry, or policy semantics into a separate source of truth.

## Not In OSS Core

- Billing, tenants, user accounts, OAuth apps, installation token storage, cloud queues, databases,
  or dashboards.
- A replacement standard for SARIF, CycloneDX, OPA, SPDX, AGENTS.md, MCP, or GitHub Code Scanning.
- A new agent execution framework competing with Codex, Claude Code, LangGraph, CrewAI, AutoGen, or
  OpenAI Agents SDK.

## Commercial Thesis

The free core creates trust and adoption by becoming a small standard component in CI. The commercial
layer monetizes operational scale: many repos, many teams, retention, audit, permissions, reporting,
and support.
