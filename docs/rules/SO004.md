# SO004: asymmetric-conflict-declaration

Severity: warning

Skill A declares `conflict_with: [B]`, but skill B does not declare A.

## Why It Matters

One-way conflict declarations are valid in Skills Orchestrator today. This warning exists because
asymmetric conflicts are harder to audit and explain in larger skill sets.

## Fix

If the conflict is truly mutual, add the reverse declaration. If the one-way declaration is
intentional, leave it as-is or configure stricter policy later.
