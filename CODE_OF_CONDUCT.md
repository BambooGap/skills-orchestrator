# Code of Conduct

## Our Standard

Skills Orchestrator is a technical project for AI instruction governance. Project spaces should
stay focused, respectful, and useful for contributors, maintainers, and adopters.

Expected behavior:

- Use welcoming and inclusive language.
- Disagree with ideas directly, not with people personally.
- Keep issues and pull requests grounded in reproducible evidence.
- Respect maintainainer time by keeping reports scoped and actionable.
- Avoid posting secrets, private customer data, exploit details, or unrelated promotional content.

Unacceptable behavior:

- Harassment, intimidation, or personal attacks.
- Sexualized language or imagery.
- Publishing private information without explicit permission.
- Deliberately derailing technical discussions.
- Sharing vulnerability details publicly instead of following `SECURITY.md`.

## Enforcement

The current maintainer may hide, edit, or remove comments and may block contributors whose behavior
is unsafe for the project. Enforcement decisions should be proportionate to impact and documented
when practical.

For private security issues, use `SECURITY.md`. For conduct issues, open a GitHub issue titled
`conduct contact request` with only a high-level summary; a maintainer will move the conversation to
a private channel if needed.
